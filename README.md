# Podkop Rule Sets

Custom rule sets for Podkop / sing-box.

These rule sets are automatically generated by GitHub Actions and published as GitHub Releases.

---

## Direct

### ru-direct.srs

Russian domains that should always bypass the proxy.

Release:

```text
https://github.com/aydar-sharifulin/podkop-rules/releases/download/latest/ru-direct.srs
```

Examples:

- `.ru`
- `.рф`
- `.su`
- manually added domains

## Block

Two OISD-based blocking rule sets are available.

### oisd_big.srs

More aggressive advertisement, tracking, telemetry and malicious-domain blocking.

Generated automatically from:

```text
https://dl.oisd.nl/oisd_big_surge.list
```

Release:

```text
https://github.com/aydar-sharifulin/podkop-rules/releases/download/latest/oisd_big.srs
```

Recommended for users who prefer maximum blocking.

---

### oisd_small.srs

Less aggressive advertisement and malicious-domain blocking with improved compatibility.

Generated automatically from:

```text
https://dl.oisd.nl/oisd_small_surge.list
```

Release:

```text
https://github.com/aydar-sharifulin/podkop-rules/releases/download/latest/oisd_small.srs
```

Recommended for home routers and mobile devices where application compatibility is important.

> Only one OISD rule set should be used at a time.

## OISD Allowlist

Both OISD rule sets are processed before compilation.

Domains listed in:

```text
allowlist/oisd.txt
```

are automatically removed from both:

- `oisd_big.srs`
- `oisd_small.srs`

This allows keeping the OISD lists up to date while excluding domains that are required by certain applications.

Current allowlist:

```text
appmetrica.yandex.net
uaas.yandex.ru
```

These exclusions were added after testing because blocking them prevented the Finuslugi iOS application from starting correctly.

To add another exclusion simply append one domain per line to:

```text
allowlist/oisd.txt
```

Example:

```text
appmetrica.yandex.net
uaas.yandex.ru
example.com
```

GitHub Actions automatically removes matching domains from every generated OISD rule set before compiling the final `.srs` files.

## Automatic Updates

GitHub Actions automatically:

- downloads external rule lists;
- generates sing-box JSON rule sets;
- applies the OISD allowlist;
- validates generated JSON files;
- verifies rule counts;
- downloads the required sing-box version;
- compiles every `rules/*.json` into `.srs`;
- publishes the latest GitHub Release.

The workflow runs:

- manually (`workflow_dispatch`);
- on repository changes;
- once per day.

Podkop can download updated rule sets automatically every day.

## Routing Policy

| Rule Set | Action |
|----------|--------|
| `ru-direct.srs` | DIRECT |
| `ip-checkers.srs` | DIRECT |
| `ru.srs` | DIRECT |
| `oisd_big.srs` or `oisd_small.srs` | BLOCK |
| everything else | PROXY |

The BLOCK rule should be evaluated before DIRECT and PROXY rules.

### Block

Maximum blocking:

```text
https://github.com/aydar-sharifulin/podkop-rules/releases/download/latest/oisd_big.srs
```

or better compatibility:

```text
https://github.com/aydar-sharifulin/podkop-rules/releases/download/latest/oisd_small.srs
```

For most home users, **`oisd_small.srs` is recommended**. It provides excellent ad and tracking protection while reducing compatibility issues with banking and government applications.

Both OISD rule sets generated by this repository automatically exclude the domains listed in `allowlist/oisd.txt`, preserving compatibility with applications such as Finuslugi while keeping the rest of the OISD protection intact.
